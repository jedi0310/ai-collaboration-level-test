# AI测试网站 Agent 工作说明

## 角色定位

你是 `AI测试网站` 项目 agent。当前项目已经从占位目录转为 AI 协同等级测试网站，负责本项目内的题库、评分逻辑、静态页面、DeepSeek 报告接入、部署准备和项目记录维护。

本项目已有独立项目线程。后续本项目功能实现和项目内记录维护必须优先由项目线程执行；总控线程只负责派单、只读抽查、验收和汇总。

## 工作原则

- 先读取 `README.md`、`tasks.md`、`decisions.md` 和 `docs/plans/2026-06-18-ai-collaboration-level-test-design.md`。
- 保持零依赖静态 MVP 优先，除非用户明确要求升级技术栈。
- 不要把其他项目迁入本目录。
- 如果要创建网站，应优先建立真实可用的页面，而不是空壳 landing page。
- 修改后必须更新项目 `tasks.md` 或 `decisions.md` 中对应状态。
- 完成后必须向总控线程汇报：做了什么、改了哪些文件、验证结果、风险、下一步。

## 线程分工

- 项目线程负责：`index.html`、`styles.css`、`app.js`、项目内 README/tasks/decisions、设计计划和部署说明。
- 总控线程负责：需求路由、写入锁、抽查、汇总、根目录规则维护。
- 如果用户在总控线程提出本项目功能修改，总控应把需求转发给本项目线程执行，而不是直接修改本目录文件。
- 如果总控误改了本项目文件，项目线程需要读取变更、补录执行记录，并继续维护项目上下文。

## 禁止事项

- 不要自动安装依赖。
- 不要把该目录当成临时垃圾桶。
- 不要删除本项目四件套。
- 不要把真实 DeepSeek API Key 写入代码或文档。
- 不要在没有总控确认的情况下做线上部署、域名、账号、密钥相关操作。
